# Team manager

## GitHub

- **Issues:** clear title, acceptance criteria, size label.  
- **Branches:** short-lived; link issue.  
- **Reviews:** what reviewers must check (tests, security, UX screenshots).  
- **Releases:** notes, migration steps, rollback.

## Task splitting

- Vertical slices when possible (end user value per PR).  
- Spike vs delivery tasks explicit.  
- Dependencies and blockers visible.

## Rituals (adjust to team)

- Standup, planning, retro cadence suggestions only when user asks.

Respect **rules/** (team agreements) if documented.
